# ADK Agent — Google Search + Gemini 3

This repository demonstrates building and running an AI agent using the Google Agent Development Kit (ADK), powered by Gemini 3 and configured with the Google Search tool.

## Prerequisites
- uv (fast Python package installer/resolver)
- A Google Cloud Project with the Gemini API enabled
- A valid GOOGLE_API_KEY

## Quick Setup

1. Initialize project
```bash
uv init
```

2. Install ADK and GenAI libraries
```bash
uv add google-adk
uv add google-genai
```

3. Configure Google API key
macOS / Linux:
```bash
export GOOGLE_API_KEY="YOUR_API_KEY"
```
Windows PowerShell:
```powershell
$Env:GOOGLE_API_KEY="YOUR_API_KEY"
```

4. Activate virtual environment
```bash
source .venv/bin/activate
```

5. Create the ADK agent scaffold
```bash
adk create my_agent
```
- During creation: choose Gemini 3 for the root agent.
- When prompted or in the generated scaffold, enable or add the Google Search tool so the agent can perform web searches.

6. Run the agent (terminal)
```bash
adk run my_agent
```

7. Launch web interface (port 8000)
```bash
adk web --port 8000
```
Open http://localhost:8000 to chat with the agent.

## Notes / Troubleshooting
- Ensure the Gemini API and any required Search APIs are enabled in your Google Cloud project.
- Verify GOOGLE_API_KEY is set in the same shell where you run adk commands.
- If the Search tool isn’t present, add/enable it in the agent config generated by `adk create` or consult ADK docs for adding tools.
- MCP requires Python 3.10 or above. Upgrade your Python version in order to use it.


That's it — the agent should now run with Gemini 3 and Google Search enabled.
